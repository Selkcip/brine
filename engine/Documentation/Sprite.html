Class: Sprite

Sprite

Simplifies the process of transforming, drawing, and updating sprites. Can be used to display images for any purpose including player characters and backgrounds. Allows hierarchical grouping of sprites which can be used to create complex characters or systems out of individual sprites.

new Sprite()

Creates a new instance of Sprite.
Properties:
Name Type Description
alpha number The transparency of the sprite. 1 is fully opaque and 0 is fully transparent.
blendFunc Object Setter/getter for sprite.blendFunction
blendFunction Object Specifies how the sprite should be blended with the canvas when drawing with webGL. Most common are: BLEND_ALPHA, BLEND_ADD, and BLEND_MULTIPLY. Custom functions can be specified as {a:"SRC_ALPHA", b:"ONE_MINUS_SRC_ALPHA"}. Other possible values can be found here http://www.opengl.org/sdk/docs/man/xhtml/glBlendFunc.xml
blendMode string Specifies how the sprite should be blended with the canvas when drawing with the 2D canvas.
children List<Sprite> A list of sprites that have been made children of this sprite.
frame number The current frame in an animated sprite.
frameCount number The number of frames in an animated sprite.
frameHeight number The height of an individual frame in an animated sprite.
frameWidth number The width of an individual frame in an animated sprite.
height number The unscaled height of the sprite.
image Image The image to be used when drawing this sprite. The easiest way to set this is: sprite.image = Textures.load("sprite.png")
index number Used to determine the order in which sprites should be drawn. The smaller the index the closer to the front the sprite will be.
multColor Vector Color to multiply the sprite by when drawing with webGL, allowing tinting. The 2D canvas does not support efficient pixel manipulation including color multiplication.
offsetX number The x position of the sprite's origin within the sprite. By default this is the top left corner.
offsetY number The y position of the sprite's origin within the sprite. By default this is the top left corner.
parent Sprite The sprite's parent sprite.
pos Vector The sprite's position relative to its parent's position.
rotation number The rotation of the sprite in radians. Use DTR(degreees) to easily convert degrees to radians.
scaleX number The amount to scale the sprite along the x axis.
scaleY number The amount to scale the sprite along the y axis.
scrollX number The distance to scroll the sprite's image within its bounds along the x axis.
scrollY number The distance to scroll the sprite's image within its bounds along the y axis.
shader Shader When drawing with webGL, if set this shader will be used to draw the sprite instead of the default shader.
sliceHeight number The height of the slice to be sampled within the sprite's image starting from sliceY. This allows drawing only portions of the image.
sliceWidth number The width of the slice to be sampled within the sprite's image starting from sliceX. This allows drawing only portions of the image.
sliceX number The x position within the sprite's image to start sampling from. This allows drawing only portions of the image.
sliceY number The y position within the sprite's image to start sampling from. This allows drawing only portions of the image.
tilesX number The number of times to tile the sprite's image within its bounds along the x axis. Allows decimal values.
tilesY number The number of times to tile the sprite's image within its bounds along the y axis. Allows decimal values.
visible bool If false the sprite will not be drawn and neither will its children. Does not affect updating.
width number The unscaled width of the sprite.
worldRotation number Setter/getter for sprite.rotation relative to the sprite's parent's rotation.
x number Setter/getter for sprite.pos.x
y number Setter/getter for sprite.pos.y
z number Setter/getter for sprite.pos.z

              Sprite

              Simplifies the process of transforming, drawing, and updating sprites. Can be used to display images for any purpose including player characters and backgrounds. Allows hierarchical grouping of sprites which can be used to create complex characters or systems out of individual sprites.

              new Sprite()

              Creates a new instance of Sprite.
              Properties:
              Name Type Description
              alpha number The transparency of the sprite. 1 is fully opaque and 0 is fully transparent.
              blendFunc Object Setter/getter for sprite.blendFunction
              blendFunction Object Specifies how the sprite should be blended with the canvas when drawing with webGL. Most common are: BLEND_ALPHA, BLEND_ADD, and BLEND_MULTIPLY. Custom functions can be specified as {a:"SRC_ALPHA", b:"ONE_MINUS_SRC_ALPHA"}. Other possible values can be found here http://www.opengl.org/sdk/docs/man/xhtml/glBlendFunc.xml
              blendMode string Specifies how the sprite should be blended with the canvas when drawing with the 2D canvas.
              children List<Sprite> A list of sprites that have been made children of this sprite.
              frame number The current frame in an animated sprite.
              frameCount number The number of frames in an animated sprite.
              frameHeight number The height of an individual frame in an animated sprite.
              frameWidth number The width of an individual frame in an animated sprite.
              height number The unscaled height of the sprite.
              image Image The image to be used when drawing this sprite. The easiest way to set this is: sprite.image = Textures.load("sprite.png")
              index number Used to determine the order in which sprites should be drawn. The smaller the index the closer to the front the sprite will be.
              multColor Vector Color to multiply the sprite by when drawing with webGL, allowing tinting. The 2D canvas does not support efficient pixel manipulation including color multiplication.
              offsetX number The x position of the sprite's origin within the sprite. By default this is the top left corner.
              offsetY number The y position of the sprite's origin within the sprite. By default this is the top left corner.
              parent Sprite The sprite's parent sprite.
              pos Vector The sprite's position relative to its parent's position.
              rotation number The rotation of the sprite in radians. Use DTR(degreees) to easily convert degrees to radians.
              scaleX number The amount to scale the sprite along the x axis.
              scaleY number The amount to scale the sprite along the y axis.
              scrollX number The distance to scroll the sprite's image within its bounds along the x axis.
              scrollY number The distance to scroll the sprite's image within its bounds along the y axis.
              shader Shader When drawing with webGL, if set this shader will be used to draw the sprite instead of the default shader.
              sliceHeight number The height of the slice to be sampled within the sprite's image starting from sliceY. This allows drawing only portions of the image.
              sliceWidth number The width of the slice to be sampled within the sprite's image starting from sliceX. This allows drawing only portions of the image.
              sliceX number The x position within the sprite's image to start sampling from. This allows drawing only portions of the image.
              sliceY number The y position within the sprite's image to start sampling from. This allows drawing only portions of the image.
              tilesX number The number of times to tile the sprite's image within its bounds along the x axis. Allows decimal values.
              tilesY number The number of times to tile the sprite's image within its bounds along the y axis. Allows decimal values.
              visible bool If false the sprite will not be drawn and neither will its children. Does not affect updating.
